# Getting Started with Next.js: A Comprehensive Guide

Next.js has become one of the most popular frameworks for building React applications. It provides a robust set of features out of the box, including server-side rendering, static site generation, API routes, and more. In this guide, we'll explore how to get started with Next.js and build modern web applications.

## Why Next.js?

Next.js offers several advantages over traditional React applications:

- **Server-Side Rendering (SSR)**: Improves performance and SEO by rendering pages on the server.
- **Static Site Generation (SSG)**: Pre-renders pages at build time for faster page loads.
- **API Routes**: Create backend functionality without a separate server.
- **File-Based Routing**: Intuitive routing based on the file system.
- **Built-in CSS and Sass Support**: Style your components with ease.
- **Image Optimization**: Automatically optimizes images for better performance.

## Setting Up a Next.js Project

Creating a new Next.js project is straightforward:

```bash
npx create-next-app@latest my-next-app
cd my-next-app
npm run dev
```

This will create a new Next.js project with the latest features and start the development server.

## Project Structure

A typical Next.js project has the following structure:

```
my-next-app/
├── app/                  # App Router (Next.js 13+)
│   ├── layout.tsx        # Root layout
│   ├── page.tsx          # Home page
│   └── about/
│       └── page.tsx      # About page
├── components/           # React components
├── public/               # Static assets
├── styles/               # CSS files
├── next.config.js        # Next.js configuration
└── package.json          # Project dependencies
```

## Key Concepts

### Pages and Routing

In Next.js, pages are created by adding files to the `app` directory. The file path determines the URL path:

- `app/page.tsx` → `/`
- `app/about/page.tsx` → `/about`
- `app/blog/[slug]/page.tsx` → `/blog/:slug` (dynamic route)

### Data Fetching

Next.js provides several methods for data fetching:

```typescript
// Server Component (default in App Router)
async function getData() {
  const res = await fetch('https://api.example.com/data', {
    next: { revalidate: 3600 },
  })
  return res.json()
}

export default async function Page() {
  const data = await getData()
  return <main>{/* Render data */}</main>
}
```

### Styling

Next.js supports various styling solutions:

```css
/* Global CSS */
body {
  margin: 0;
  padding: 0;
  font-family: sans-serif;
}
```

```typescript
// CSS Modules
import styles from './Button.module.css'

export function Button() {
  return <button className={styles.button}>Click me</button>
}
```

## Deployment

Deploying a Next.js application is simple with platforms like Vercel:

```bash
npm run build
npm run start
```

Or deploy directly to Vercel:

```bash
vercel
```
